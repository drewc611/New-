# Writing & code style (anti-AI rules)

## Tone for user-facing copy
- Plain, operator-speak English. No "delve", "leverage", "seamless",
  "streamline", "furthermore", "it's worth noting", "unlock", "empower".
- No em-dash-padded clauses — they read as AI tics here.
- Short sentences. One idea per line. No hedging throughout.
- Units and numbers: always concrete ("21,242 records", not "many records").

## Code
- TypeScript is strict — no `any`, no unused imports, no unused locals.
- Tailwind tokens only; no arbitrary `w-[...]` values unless they're in the
  compiled CSS.
- Icons come from `lucide-react`. Don't inline random SVGs except for the
  USPS eagle mark.
- React: functional components, hooks at the top, no class components.
- Zustand for shared state. No Redux, no Context towers.

## Comments
- Default: none. Code + names tell the story.
- If you must comment: explain the *why* — a constraint, a workaround, a
  non-obvious invariant. Never restate the line above it.
- No author tags, no "added for ticket X" — PR desc holds that.

## Commit messages
- Imperative, past-tense-free. First line ≤ 70 chars.
- Body explains the *why*; never a diff recap.
- No emoji. No trailing "Generated by …" lines beyond the session URL the
  tool appends.
