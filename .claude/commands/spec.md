Before any code, draft a spec for: $ARGUMENTS

1. If scope is ambiguous, ask up to 5 clarifying questions and wait.
2. Copy `cowork/templates/spec.yaml` to `cowork/outputs/<kebab-name>.yaml`.
3. Fill in `goal`, `decisions` (the WHY), `steps` (implement / verify), `done`.
4. Show the filled spec. Wait for approval before implementing.

Principles: avoid vibe coding. Spec first, code second. Document the why, not the what.
