Thanks for installing {{ .Chart.Name }} v{{ .Chart.AppVersion }}.

Release: {{ .Release.Name }}
Namespace: {{ .Release.Namespace }}
LLM provider: {{ .Values.llm.provider }}
Address verifier: {{ .Values.addressVerifier.provider }}
Auth enabled: {{ .Values.auth.enabled }}

1. Check that the pods are running:

   kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

{{- if .Values.ingress.enabled }}

2. Reach AMIE via the ingress host:
{{- range .Values.ingress.hosts }}
   http{{ if $.Values.ingress.tls }}s{{ end }}://{{ .host }}/
{{- end }}

{{- else }}

2. Port forward the frontend service to reach AMIE locally:

   kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "amie.frontendServiceName" . }} 8080:{{ .Values.frontend.service.port }}
   open http://localhost:8080
{{- end }}

3. Backend health check:

   kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "amie.backendServiceName" . }} 8000:{{ .Values.backend.service.port }}
   curl http://localhost:8000/api/health

{{- if .Values.ollama.enabled }}

4. Pull an Ollama model so the chatbot has something to serve:

   kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "amie.fullname" . }}-ollama -- ollama pull {{ .Values.llm.ollamaModel }}
{{- end }}

5. First time users: the vector index is built automatically from the packaged
   knowledge base on backend startup. Check logs if searches return empty:

   kubectl logs -n {{ .Release.Namespace }} deploy/{{ include "amie.fullname" . }}-backend --tail 100
